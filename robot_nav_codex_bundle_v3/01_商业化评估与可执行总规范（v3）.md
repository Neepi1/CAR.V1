# 室内外一体配送机器人导航系统：商业化评估与可执行总规范（v3）

版本：v3.0  
目标：为 Codex 提供一份可直接执行的、多楼层、离线建图与在线导航分离、优先复用本地 car 工程资产的自动化规格。  
目标平台：Jetson Orin + ROS 2 Humble + JT128 + Ranger Mini 3 + FAST-LIO2 + PGO + Isaac ROS Occupancy Grid Localizer + Nav2。

---

## 1. 商业化判断

### 1.1 当前结论
本版可作为：
- 楼宇 / 园区场景的 **Beta 试运营基线**
- 固定路线 + 多楼层 + 电梯配送的 **商业试点软件栈**

本版仍不是“量产最终版”，还需要：
- 真实底盘/车体参数固化
- 电梯协议接入
- 现场安全验收
- 日志回传与运维
- OTA / 版本管理

### 1.2 当前已确认可用的已有链路
- JT128 驱动与网络已跑通
- 传感器安装外参已有可用版本
- FAST-LIO2 已跑通
- PGO 已跑通
- Isaac ROS Occupancy Grid Localizer 已跑通
- 主要工程问题集中在 TF 树治理、组件包装和资产化

## 2. 绝对固定的技术决策
Codex 不得擅自更改以下默认实现：

- ROS 2: **Humble**
- DDS: **Fast DDS (`rmw_fastrtps_cpp`)**
- 离线建图前端：**FAST-LIO2**
- 离线建图后端：**PGO / Loop Closure**
- 离线建图标准输出：`optimized_map.pcd + optimized_trajectory.csv + metadata.yaml`
- 在线连续 odom 默认：**wheel odom + JT128 IMU -> robot_localization EKF**
- 在线全局定位：**Isaac ROS Occupancy Grid Localizer**
- `map -> odom` 桥接：**robot_localization_bridge**
- 默认 global planner：**SmacPlanner2D**
- 默认 local controller：**MPPI**
- fallback controller：**RPP**
- path smoother：**Simple Smoother**
- velocity smoother：**必须启用**
- collision monitor：**必须启用**
- local costmap obstacle source：**只允许来自 `robot_local_perception`**
- 多楼层：**一层一图 + floor_manager 原子切图 + 电梯厅重定位**

## 4. 最终 TF 与位姿架构

### 4.1 最终导航主树
```text
map
 └── odom                      (dynamic, only robot_localization_bridge)
      └── base_link            (dynamic, only robot_local_state)
           ├── lidar_link      (static)
           ├── imu_link        (static)
           ├── base_footprint  (optional static)
           └── other static frames
```

### 4.2 角色定义
- `robot_local_state`：唯一提供连续局部运动估计，发布 `odom -> base_link`
- `robot_global_localization`：输出 `base_link` 相对 `map` 的全局姿态结果，不直接抢占 `odom -> base_link`
- `robot_localization_bridge`：计算并发布 `map -> odom`
- `robot_description` / `robot_state_publisher`：发布静态传感器外参

### 4.3 禁止项
- 任何重复的 `odom -> base_link`
- 任何重复的 `map -> odom`
- 任意第三方裸节点直接把内部算法 frame 当导航主 frame
- 重复静态外参发布

## 5. 离线建图链路（正式版）

```text
JT128 点云 + JT128 IMU
  -> robot_hesai_jt128
  -> robot_fastlio_mapping (FAST-LIO2 frontend wrapper)
  -> robot_pgo_mapping (loop closure / backend wrapper)
  -> mapping_result/
       optimized_map.pcd
       optimized_trajectory.csv
       metadata.yaml
       loop_report.json
  -> robot_map_toolkit.import
  -> pointcloud_clean
  -> floor_split
  -> traversability
  -> occupancy_intermediate
  -> nav_map export (yaml + pgm)
  -> localizer_map export (png + params)
  -> filter masks export
  -> asset validation
```

### 5.1 输出契约
每层必须导出：
```text
maps/building_X/floor_k/
  nav/
    nav_map.yaml
    nav_map.pgm
  localizer/
    localizer_map.png
    localizer_params.yaml
  filters/
    keepout_mask.yaml
    keepout_mask.pgm
    speed_mask.yaml
    speed_mask.pgm
    binary_mask.yaml
    binary_mask.pgm
  reports/
    asset_report.json
  poses.yaml
```

### 5.2 PGO 固定要求
- 必须保留回环报告
- 必须能在离线 bag 上复跑
- 必须能在不改 FAST-LIO2 核心源码的前提下完成 wrapper 集成
- 不允许把“只有 FAST-LIO2 前端，没有回环后端”的链路当默认量产地图生产线

## 6. 在线导航链路（正式版）

```text
底盘驱动 -> robot_chassis_bridge -> /wheel/odom + /cmd_vel feedback + diagnostics
JT128 驱动 -> robot_hesai_jt128 -> /sensors/lidar/points_raw + /sensors/lidar/imu

/wheel/odom + /sensors/lidar/imu
  -> robot_local_state (robot_localization EKF)
  -> /local_state/odometry
  -> odom->base_link

/sensors/lidar/points_raw + mode state
  -> robot_local_perception
  -> /perception/obstacle_points

/sensors/lidar/points_raw
  -> robot_global_localization
      - pointcloud_to_flatscan
      - isaac occupancy grid localizer
      - trigger localization
      - publish /global_localization/pose
      - publish /global_localization/health
      - apply floor localizer assets

/global_localization/pose + /local_state/odometry
  -> robot_localization_bridge
  -> map->odom
  -> /localization/health

Nav2
  planner_server (SmacPlanner2D)
  smoother_server (Simple Smoother)
  controller_server (MPPI)
  behavior_server
  bt_navigator
  velocity_smoother
  collision_monitor
  lifecycle_manager_localization
  lifecycle_manager_navigation

controller -> velocity_smoother -> collision_monitor -> robot_safety -> robot_chassis_bridge
```

### 8.1 新增或收紧的包职责
#### robot_fastlio_mapping
- 只负责 FAST-LIO2 wrapper
- 统一输入输出 topic / 文件契约
- 负责关闭或隔离不应进入导航主树的 TF
- 输出 `frontend_result` 给 `robot_pgo_mapping`

#### robot_pgo_mapping
- 只负责回环 / PGO wrapper
- 消费 `frontend_result`
- 输出最终 `mapping_result`
- 记录 `loop_report.json`

#### robot_global_localization
- 统一封装 PointCloud -> FlatScan -> Isaac localizer
- 提供 floor asset apply / reload / restart
- 监控 localizer health
- 明确输出 global pose，不直接发布重复 TF

#### robot_localization_bridge
- 计算 `map -> odom`
- 实现 jump suppression / timeout gating / health state
- 不能直接覆盖 `odom -> base_link`

#### robot_local_perception
- 不能把原始 JT128 点云直接喂给 local costmap
- 必须输出经过过滤的 obstacle cloud
- 至少支持 NORMAL / RAMP / ELEVATOR_WAIT / DOORWAY 四种模式

## 9. 多楼层与电梯
- 一层一图
- `map_server` 只加载当前楼层 nav map
- `robot_global_localization` 只加载当前楼层 localizer png + params
- 切楼层时必须：停车 -> 切图 -> 切 localizer 资产 -> 触发重定位 -> 恢复 mission
- 不允许 planner 直接跨楼层自由规划

---

## 10. Nav2 固定实现
### 10.1 Planner
- 默认：SmacPlanner2D
- 可选 profile：SmacHybrid（仅在你明确要求最小转弯半径约束时启用）

### 10.2 Controller
- 默认：MPPI
- Fallback：RPP
- 禁止把 DWB 作为默认

### 10.3 Costmaps
#### Global costmap
- StaticLayer
- InflationLayer
- Costmap Filters（keepout / speed / binary）

#### Local costmap
- VoxelLayer
- InflationLayer
- obstacle source 只允许 `/perception/obstacle_points`

### 10.4 Safety chain
必须固定为：
```text
controller_server
  -> /cmd_vel_nav_raw
  -> velocity_smoother
  -> /cmd_vel_nav
  -> collision_monitor
  -> /cmd_vel_collision_checked
  -> robot_safety
  -> /cmd_vel
  -> robot_chassis_bridge
```
