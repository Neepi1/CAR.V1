# 新增关键要求与 TF 修复优先级（v3）

本文件是在 v2 基础上新增的强制要求，优先级高于旧版描述。

## 0. 当前已知事实
- JT128 驱动已跑通。
- JT128 网络接口已跑通。
- 传感器安装外参已存在可用版本。
- FAST-LIO2 已跑通。
- PGO / 回环链路已跑通。
- Isaac ROS Occupancy Grid Localizer 已跑通。
- 当前核心阻塞不是“组件不存在”，而是 **TF 树不正确 / 不稳定 / 不一致**。

## 1. 最高优先级目标
在继续搭系统前，必须先把 TF 树修正到可解释、可回放、可重定位的状态。

### 1.1 目标 TF 树
最终在线运行 TF 树必须收敛到：

```text
map
 └── odom                      (dynamic, from robot_localization_bridge)
      └── base_link            (dynamic, from robot_local_state)
           ├── lidar_link      (static)
           ├── imu_link        (static)
           ├── base_footprint  (optional static)
           └── other static frames
```

### 1.2 明确禁止的情况
- 禁止同时有多个节点发布 `odom -> base_link`。
- 禁止同时有多个节点发布 `map -> odom`。
- 禁止 JT128 驱动、FAST-LIO2、PGO、Isaac localizer、底盘驱动各自乱发重复 TF。
- 禁止把 `wheel_odom`、`camera_init`、`aft_mapped` 之类算法内部 frame 直接暴露为导航主树的一部分。
- 禁止让 `base_link -> lidar_link`、`base_link -> imu_link` 既在 URDF 里发布又在 static_transform_publisher 重复发布。

## 2. car 项目复用优先级（强制）
Codex 必须优先复用你现有 car 项目中的参数、launch、依赖与第三方源码，不得一上来联网重装。

### 2.1 复用顺序
1. 扫描当前仓库 / 工作区 / 上级目录中的 car 项目。
2. 优先复用 car 项目中以下内容：
   - JT128 驱动参数
   - 网卡配置 / IP / 端口
   - 安装外参
   - FAST-LIO2 参数
   - PGO 参数
   - Isaac localizer 参数
   - 已验证可用的第三方源码版本
   - docker/devcontainer/安装脚本
3. 只有在本地不存在、或经验证不兼容时，才允许联网下载。

### 2.2 需要主动扫描的路径模式
Codex 必须在以下模式中搜索：
- `./car`
- `./car_project`
- `./src/car*`
- `../car`
- `../car_project`
- 用户在 README / docs / shell 脚本中引用的外部工作区路径

### 2.3 必须输出的复用报告
Codex 必须生成 `reports/car_project_reuse_report.md`，列出：
- 找到了哪些本地参数/launch/依赖
- 哪些被直接复用
- 哪些被包装后复用
- 哪些本地没有，因而需要联网下载

## 3. PGO / 回环现在是正式组件
从 v3 开始，PGO / 回环不是可选项，而是正式离线建图后端。

### 3.1 离线建图正式链路
```text
JT128 点云 + JT128 IMU
  -> FAST-LIO2 (frontend)
  -> PGO / loop closure (backend)
  -> optimized_map.pcd
  -> optimized_trajectory.csv
  -> metadata.yaml
  -> robot_map_toolkit
```

### 3.2 输出契约
离线建图最终标准输出必须为：
- `mapping_result/optimized_map.pcd`
- `mapping_result/optimized_trajectory.csv`
- `mapping_result/metadata.yaml`
- `mapping_result/loop_report.json`

## 4. 现阶段必须先做的事情
### P0：TF 审计
- 枚举当前所有 TF 发布者
- 保存 `view_frames` 输出
- 记录重复发布与冲突 frame
- 记录每个动态 TF 的唯一来源

### P1：把“已跑通”组件包起来
对下列“已跑通但可能直接裸跑”的组件必须做 wrapper：
- JT128 driver wrapper
- FAST-LIO2 wrapper
- PGO wrapper
- Isaac localizer wrapper

### P2：先打通复用
Codex 必须先尝试复用 car 项目参数/依赖，复用失败才下载。

## 5. 运行期最终权责划分
- `robot_local_state`: 唯一发布 `odom -> base_link`
- `robot_localization_bridge`: 唯一发布 `map -> odom`
- `robot_description` / `robot_state_publisher`: 唯一发布静态传感器外参 TF
- FAST-LIO2 / PGO / Isaac localizer: 默认不直接向导航主树发布重复 TF；如第三方节点无法关闭 TF，则必须在 wrapper 中拦截/替换/参数关闭

## 6. 这版的核心变化
相对 v2，本版新增两条硬约束：
1. **PGO / 回环正式纳入离线建图主链路**
2. **本地 car 项目参数与依赖优先复用，缺失时才允许联网下载**
